搭建项目
Angular cli是一个命令行界面工具，它可以创建项目、添加文件以及执行一大堆开发任务，比如测试、打包和发布，这里的项目搭建就是基于这个命令。
开始项目前，你需要先安装node和npm，然后执行npm install -g @angular/cli安装Angular CLI。
先执行下面的命令初始化项目（其中--skip-install表示node包先不安装，接着使用cnpm install安装会快多了）：
ng new newApp --skip-install
接着进入项目文件夹，安装node包：
cnpm install
启动项目（浏览器会自动打开页面）：
ng serve --open
此时，项目就搭建成功了。
目录结构
现在来看看ng命令帮助我们生成了什么，也就是项目的目录结构，里面都是干什么的，先有个大致了解，你可以不知道全部，不过先记住下面几个个人感觉重要的：
  • 1.src：应用代码存放的地方；
  • 2.src/app：你的代码主要存放的地方，这样说也许不合适，不过你开发的时候，大部分时间都是在修改这里的代码；
  • 3.src/assets：图片等存放的地方，构建时会复制到发布包里；
  • 4.src/main.js：你基本不会修改它，它是程序的主入口；
  • 5.src/styles.css：特别用的样式写在对应的地方，后面会说，对于公共的样式就会写在这里；
  • 6.karma.conf.js：给Karma的单元测试配置，当运行ng test时会用到它。
app.module.ts
上面描述了主要的文件意义，接着，来看一下模块文件src/app/app.module.ts文件，其中定义了AppModule，根模块为Angular描述如何组装应用，先来看一下里面的内容： 
import { BrowserModule } from '@angular/platform-browser';
import { NgModule } from '@angular/core';
            
import { AppComponent } from './app.component';
            
@NgModule({
    declarations: [
        AppComponent
    ],
    imports: [
        BrowserModule
    ],
    providers: [],
    bootstrap: [AppComponent]
})
export class AppModule { }
注意里面的bootstrap属性设置了[AppComponent]，表示启动组件，以后添加新的组件或者服务、指令等就可以在里面嵌套添加或被调用了（当然，除了局部使用的服务，别的都需要在这里引入或声明）。
