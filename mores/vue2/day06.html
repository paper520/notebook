基本使用方法
通过prop可以实现父组件传递信息给子组件，比如现在定义了一个子组件prop.component.js，代码如下： 
import Vue from 'vue';
            
Vue.component('prop-comp',{

    props:['info'],

    template:'<em>{{info}}</em>'

});
使用的时候，在父组件就可以通过info属性来传递数据给子组件，代码如下：
<prop-comp info="这是父组件传递给子组件的信息"></prop-comp>
大小写问题
需要注意prop的大小写问题，由于HTML对于大小写不敏感，因此比如你在js里面定义了如下prop：
props:['userInfo']
在页面中使用的时候，应该这样使用：
<prop-comp user-info="这是父组件传递给子组件的信息"></prop-comp>
当然，如果你使用了字符串模板，因为不是HTML中，这个限制就没有了。
定义值类型
在前面的例子中，都是通过字符串类型的数组来说明需要传递的数据，比如：
props:['propA','propB','propC']
其实，这些传递的值的类型，也是可以指定的，下面是例子代码： 
props: {
    
    // 基础的类型检查 ('null'匹配任何类型)
    propA: Number,

    // 多个可能的类型
    propB: [String, Number],

    // 必填的字符串
    propC: {
        type: String,
        required: true
    },
                
    // 带有默认值的数字
    propD: {
        type: Number,
        default: 100
    },
                
    // 带有默认值的对象
    propE: {
        type: Object,
        // 对象或数组且一定会从一个工厂函数返回默认值
        default: function () {
            return { message: 'hello' };
        }
    },

    // 自定义验证函数
    propF: {
        validator: function (value) {
            // 这个值必须匹配下列字符串中的一个
            return ['success', 'warning', 'danger'].indexOf(value) !== -1;
        }
    }

}
注意那些prop会在一个组件实例创建之前进行验证，所以实例的属性(如 data、computed等)在default或validator函数中是不可用的。
